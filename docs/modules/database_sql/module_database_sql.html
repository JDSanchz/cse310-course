Module—Relational Database with SQL

Overview

Data can be stored in many different formats. An efficient method for storing information is in a database. One common type of database is a relational database, which organizes the data into tables and relates them together using keys. Libraries have been written to make it easy for the programmer to read and write to and from these databases. Most relational databases support Structured Query Language, or SQL. SQL is a language that allows the programmer to easily interact with the database. Examples of relational databases include the following: Oracle, MySQL, Microsoft SQL Server, PostgreSQL, SQLite, and IBM DB2.

You can read more about relational databases and SQL at sites including the following:

  • Relational Databases - Wikipedia (open collaboration site)

  • Relational Databases - Oracle (company sponsored site)

  • SQL - W3Schools (tutorial site)

Module Requirements

To complete this module, you need to do the following:

  • Write a program that interacts with a relational database using SQL. The purpose of the program and the type of data stored are up to you. The program can be written in any programming language and use any relational database. See the notes below for guidance about this step.

  • Create a database for your program to use, including at least one table to store the data.

  • The program should demonstrate the following capabilities working with the database:

    • Insert new data into a table.

    • Modify existing data in the table.

    • Delete existing data from the table.

    • Retrieve (that is, query) data from the table.

  • The program needs to be published to a public GitHub repository as part of your personal programming portfolio. Be careful not to include any passwords or keys in your published material. Ensure that the repository has a good description and is tagged appropriately. The README.md file in your repository should be well-documented, with at least the following information:

    • A description of your program.

    • How to set up the development environment, including the database. Provide detailed steps so that you or another engineer could replicate the environment.

    • How to build (if applicable), execute, and use your program.

    • Screenshots of your software running.

    • Description of the SQL commands you used in the program.

    • A list of useful websites that you found to assist your development work.

Guidance

You should consider using Python and PostgreSQL for this module. Here are some useful notes when using this suggested environment:

  • You need to install PostgreSQL on your computer. Installation instructions can be found on the PostgreSQL website.

  • When you install PostgreSQL, you will be prompted to create an administrator password. It is very important that you save this password so you can access the SQL Shell for setup and debugging. The SQL Shell (also called psql) is installed with PostgreSQL. The SQL Shell supports SQL commands as well as specific commands (they start with a backward slash \) to help you navigate the database. You should refer to the PostgreSQL website or one of the tutorials below to learn how to use these commands.

  • Use psycopg to access a PostgreSQL database from Python. You can use pip to install: pip install psycopg2

Here is a list of websites that you might find useful when working in the suggested environment:

  • PostgreSQL (official site)

  • psycopg (official site)

  • TutorialsPoint - PostgreSQL (tutorial site)

  • PostgreSQL tutorial using psycopg (tutorial site)
