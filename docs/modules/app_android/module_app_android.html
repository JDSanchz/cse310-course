Discovery Module - Creating an Android App

Overview

Did you know that there are close to three million applications in the Google Play Store? Did you also know that the average Android developer makes well over 100k/year? Android is everywhere, and the ability to make Android applications has never been more sought after or desirable as it is now. Android applications are typically written in Java using Android Studio, and while many applications are published to the Google Play Store or other application markets, developers are able to create applications and distribute them without such deployments. This module is intended for you to learn how to develop Android applications quickly using Flutter.

You can read more from sites such as the following:

  • Android Apps

  • Android Studio

  • Flutter: build beautiful, natively compiled mobile apps

Module Requirements

To complete this module, you need to do the following:

  • Create a native android application using Android Studio

  • This application should:

    • be deployed as an apk

    • run successfully on an Android device

    • be helpful in performing a task that you, or someone you know, do on a frequent basis

  • The program needs to be published to a public GitHub repository as part of your personal programming portfolio. Be careful not to include any passwords or keys in your published material. Ensure that the repository has a good description and is tagged appropriately. The README.md file in your repository should be well-documented with at least the following information:

    • A description of your program.

    • How to set up the development environment including all tools. Provide detailed steps so that you or another engineer could replicate the environment.

    • How to build (if applicable), execute, and use your program.

    • Screenshots of your software running.

    • A list of useful websites that you found to assist your development work.

Stretch Challenges

You can earn additional bonus points by completing one of the following stretch challenges:

  • Give your application a professional look and feel

  • Use something in Android that requires permission (for example: location, camera, microphone, etc.)

  • Store data either in local storage or in Firebase

The implementation of a stretch challenge must be done within the same GitHub repository, and reflected in the descriptions found in your README.md file.
